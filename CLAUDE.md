# Choonsikmom Archive — 게시글 작성 규칙

이 저장소의 게시글을 통일된 형식으로 작성하기 위한 규칙이다. 새 글을 쓰거나 기존 글을 고칠 때 이 규칙을 따른다.

## 1. 문체

- **서술 문장은 `-다` 평서체**로 간결하게 끝낸다. 정중체(`~합니다`, `~입니다`)를 쓰지 않는다.
  - `표현됩니다` → `표현된다`
  - `필요합니다` → `필요하다`
  - `~라고 할 수 있습니다` → `~라고 할 수 있다`
- **번호·글머리(리스트) 항목은 명사·구로 끝내도 된다.** 문장으로 끝나지 않고 단어/구로 끝나는 현재 형식을 유지한다.
  - 예: `- 위치(Translation) 3자유도`, `- 서브픽셀 정확도의 코너 검출`
- 군더더기 없이 핵심만. 한 문단은 짧게.

## 2. Front matter

```yaml
---
title: "[공부] 제목"          # 또는 [논문리뷰]
excerpt: "한 줄 요약"
categories:
  - 공부                      # 또는 논문리뷰 (메인 카테고리)
  - CV                        # 세부 분야: AI / CV / NLP / RL / Robotics
tags:
  - 키워드1
  - 키워드2
toc: true
toc_sticky: true
---
```

- **파일명**: `YYYY-MM-DD-slug.md` (slug은 영문 소문자-하이픈).
- **`date:`에 시각(시:분)을 넣지 않는다.** 시각을 넣으면 현재보다 미래일 때 Jekyll이 글을 숨긴다. 시리즈 순서는 날짜(일 단위)로 조절한다.
- 첫 카테고리가 메뉴 분류(공부/논문리뷰), 둘째가 세부 분야(사이드바 서브카테고리)를 결정한다.

## 3. 글 구조

1. 인용문(`>`) 한두 줄로 리드 — 이 글이 무엇인지/시리즈 위치
2. 번호 붙인 섹션(`## 1.`, `## 2.` …)
3. 필요하면 비교 **표**로 정리
4. 마지막에 `## 정리` + 다음 글 링크

## 4. 내부 링크 (시리즈)

- 다른 글로 링크할 때 `{% raw %}{% post_url YYYY-MM-DD-slug %}{% endraw %}` 를 쓴다.
- 시리즈 글은 **끝에 다음 글 링크**를 단다. 날짜는 읽는 순서대로 오름차순(1편이 가장 과거)으로 둬서 시간 순서와 읽는 순서를 맞춘다.

## 5. 수식

- 인라인 `$ ... $`, 블록 `$$ ... $$` (MathJax로 렌더).

## 6. 그림 (SVG 다이어그램)

개념 이해를 돕는 **SVG 다이어그램**을 적극적으로 넣는다. `assets/images/`에 `.svg`로 저장하고 마크다운으로 삽입한다.

```markdown
![설명 텍스트](/assets/images/diagram-name.svg)
```

### SVG 스타일 가이드 (통일성)

- 밝은 배경 기준, 깔끔한 sans-serif. `font-family="Pretendard, 'Malgun Gothic', sans-serif"`
- **헤더 pill**(둥근 사각형)로 항목을 구분하고, 파스텔 색으로 구별한다.
  - 파랑 계열: 배경 `#e9f1fb` / 테두리 `#b9d0ec` / 글자 `#2c5aa0`
  - 주황 계열: 배경 `#fbeed6` / 테두리 `#e6cfa0` / 글자 `#b5761a`
  - 초록 계열: 배경 `#e6f4ea` / 테두리 `#b7dcc0` / 글자 `#2e7d46`
- 내용 박스는 **점선 둥근 사각형**(`stroke:#c9ccd1; stroke-dasharray:5 4; fill:none`).
- 흐름은 **회색 화살표**(`stroke:#8a9199`)로 연결한다.
- 본문 글자색 `#333`, 보조 설명은 회색 `#6b7178`.
- `viewBox`를 지정해 반응형으로 스케일되게 한다.

> 참고 기준: TSDF vs 3DGS 비교처럼, 두 개념을 좌우로 나눠 헤더 pill + 점선 박스 + 화살표로 구성하는 형식.
